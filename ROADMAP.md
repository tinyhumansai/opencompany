# Roadmap

## Now

- Keep the Rust/Axum surface small and compile-tested.
- Define stable app, server, OpenHuman launcher, and TinyAgents status surfaces.
- Maintain docs that match the source tree.

## Next

- Add richer OpenHuman process management and health checks.
- Add API routes that expose company-specific operating workflows.
- Compile and exercise `cargo check --features tiny` in CI once submodule
  update time is acceptable.

## Later

- Add persistent company context backed by OpenHuman memory/runtime surfaces.
- Add live OpenHuman startup and TinyAgents graph examples.
- Publish generated API documentation and release artifacts.
