# Engineering standards

The bar every change holds to.

| Do | Don't |
| --- | --- |
| Ship behind a test | Merge on faith |
| Handle the error path | Assume the happy path |
| Keep changes small | Bundle unrelated work |
| Park releases for approval | Ship without sign-off |

Sensitive changes get a security review; every release waits on the operator's
approval. Applies to the [[Release checklist]] and features like [[Billing v2]].
