# Release checklist

The standard steps for taking a feature from request to ship.

1. **Spec** — scope the request into a buildable spec
2. **Build** — design, backend, and frontend against [[Engineering standards]]
3. **Review** — code review and, when sensitive, a security review
4. **QA** — test end to end and document the change
5. **Ship** — release behind the operator's approval, then report back

See [[Billing v2]] for the current run.
