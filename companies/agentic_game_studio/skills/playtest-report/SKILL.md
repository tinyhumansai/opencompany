# Playtest Report

Run a playtest and turn what you saw into changes that make the game better,
ranked by impact.

## When to use

- A build is playable and needs feedback before it advances.
- Difficulty or feel is in question and needs evidence.

## Steps

1. **Set** what this playtest is testing — a mechanic, a level, the onboarding.
2. **Observe** where players hesitate, fail, or disengage.
3. **Capture** bugs alongside feel and difficulty notes.
4. **Separate** signal from one-off preference.
5. **Rank** the findings by impact on the experience.

## Output

A report: what was tested, ranked findings on feel and difficulty, and a bug
list, following the [[Design pillars]]. Feeds the next iteration.
