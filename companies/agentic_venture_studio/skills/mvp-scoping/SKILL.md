# MVP Scoping

Define the smallest thing you can build that actually tests the bet — and cut
everything that does not.

## When to use

- A venture thesis is ready and needs a first build.
- Scope is creeping and the MVP is drifting into a v1.

## Steps

1. **Restate** the one assumption the MVP must test.
2. **List** only the capabilities that test it; defer the rest.
3. **Choose** the fastest path to a real user experience.
4. **Define** the metric that will read the result.
5. **Set** a hard timebox and what "done" means.

## Output

An MVP scope: the assumption, the minimal capability list, the success metric,
and a timebox, following the [[Studio standards]]. Ready to build.
