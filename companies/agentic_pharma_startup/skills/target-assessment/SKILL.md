# Target Assessment

Judge whether a target is worth a program — how druggable it is, and what could
kill it — before resources go in.

## When to use

- A target is proposed and needs a rigorous read before committing.
- Two targets compete for the same program slot.

## Steps

1. **Establish** the biological rationale linking target to disease.
2. **Assess** druggability — structure, binding sites, and modality fit.
3. **Scan** the competitive and IP landscape around the target.
4. **Surface** the risks: safety liabilities and validation gaps.
5. **Weigh** it into a go / no-go with the reasoning.

## Output

A target assessment: rationale, druggability, landscape, risks, and a go /
no-go, following the [[Research standards]]. The operator decides to commit.
